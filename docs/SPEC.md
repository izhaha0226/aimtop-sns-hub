## 2. 사용자 및 권한

### 2-1. 역할(권한) 정의

| 역할 | 설명 | 부여 가능 권한 |
|---|---|---|
| **Admin** | 시스템 전체 관리자 | 모든 권한 (담당자 관리 포함) |
| **Approver** | 콘텐츠 검토/승인자 | 승인 큐 + 발행 + 콘텐츠 작성 |
| **Editor** | 콘텐츠 작성 담당자 | 콘텐츠 작성 + 임시저장 + 승인 요청 |
| **Viewer** | 조회 전용 | 대시보드/분석 조회만 가능 |

> 역할은 고정 단계가 아니라 **Admin이 언제든 수정 가능**한 유동 권한입니다.

#### 담당자 정보 항목
| 필드 | 설명 |
|---|---|
| 이름 | 실명 |
| 이메일 | 로그인 ID 겸용 |

| 계정 상태 | 활성 / 비활성 (퇴사 등) |

#### 담당자 관리 기능 (FR-05)

**등록 시:**
- Admin이 이름/이메일/역할(권한) 입력 후 초대 이메일 발송
- 초대 링크에서 본인이 비밀번호 설정 후 활성화
- **등록 시점에 역할(Admin / Approver / Editor / Viewer) 반드시 지정**

**권한 수정:**
- Admin 전용 [설정 > 담당자 관리] 화면에서 언제든 역할 변경 가능
- 변경 즉시 적용 (재로그인 불필요)
- 변경 이력 자동 기록 (누가 언제 어떤 권한으로 변경했는지)

**계정 관리:**
- 비활성화 시 즉시 로그인 차단 (데이터는 보존)
- 재활성화 가능 (퇴사 후 재입사 등)
- 담당 클라이언트 할당/해제 (Admin 권한)
- 담당자별 활동 로그 조회 (발행 이력, 승인 이력, 로그인 이력, 권한 변경 이력)

### 2-4. 워크스페이스 구조

### 3-2. 채널 선택형 온보딩 (운영 채널 설정)

> 클라이언트마다 운영할 채널이 다릅니다.
> 워크스페이스 생성 시 **운영할 채널을 선택**하고, 선택된 채널에 맞게 UI/기능이 최적화됩니다.

**채널 선택 화면:**
```
운영할 SNS 채널을 선택해주세요 (복수 선택 가능)

  ☑ 인스타그램   ☑ 페이스북   ☑ X (트위터)
  ☑ Threads      ☑ 카카오채널 ☑ 틱톡
  ☐ 링크드인     ☐ 유튜브

  나중에 채널 추가/제거도 가능합니다.
```

**채널 선택 후 자동 적용:**
- 선택한 채널만 대시보드에 표시
- 게시물 작성 시 선택 채널만 발행 대상으로 노출
- 채널별 글자 수 제한 / 이미지 비율 / 최적 형식 자동 적용
- 선택 안 한 채널 관련 기능 UI에서 숨김

### 3-3. 채널별 우수 트렌드 계정 자동 벤치마킹

> 온보딩 Step 3에서 직접 입력하는 벤치마킹 외에,
> **각 채널별 업종 트렌드 상위 계정을 AI가 자동 분석**하여 콘텐츠 전략에 반영합니다.

**자동 벤치마킹 플로우:**
```
채널 선택 + 계정 유형 선택 완료
          │
          ▼
AI가 각 채널별 트렌드 상위 계정 자동 발굴
─────────────────────────────────────────
  인스타그램: 업종 해시태그 상위 계정 Top 5
  틱톡: 업종 카테고리 인기 계정 Top 5
  링크드인: 업종 팔로워 성장 계정 Top 5
  X: 업종 참여율 상위 계정 Top 5
  etc...
─────────────────────────────────────────
          │
          ▼
각 채널별 트렌드 계정 최근 30개 게시물 분석
  • 콘텐츠 유형 믹스 (카드뉴스/릴스/텍스트 비율)
  • 발행 빈도 / 최적 시간대
  • 해시태그 전략
  • 평균 참여율
  • 가장 많이 쓰는 포맷/스타일
          │
          ▼
채널별 트렌드 인사이트 리포트 생성
          │
          ▼
운영 전략서에 자동 반영
(채널별 콘텐츠 방향 / 형식 / 주기 설정)
```

**채널별 트렌드 분석 결과 예시:**

### 3-3-B. 채널 계정 인증 및 토큰 관리

---

#### 인증 방식: OAuth 2.0 (계정/비번 절대 수집 안 함)

> 클라이언트는 아이디/비번을 에임탑에 줄 필요 없습니다.
> 본인이 직접 플랫폼 로그인 후 권한만 허용하면 됩니다.

**연동 플로우:**
```
[SNS Hub 설정] "채널 연동" 버튼 클릭
        │
        ▼
플랫폼 공식 로그인 팝업 오픈 (OAuth)
클라이언트가 본인 계정 로그인
"게시물 발행 / 댓글 관리 / 인사이트 조회 허용" 클릭
        │
        ▼
SNS Hub가 Access Token + Refresh Token 수신
→ AES-256 암호화 후 DB 저장
→ 연동 완료 (클라이언트 비번은 서버에 저장 안 됨)
```

---

| 채널 | Access Token 유효기간 | Refresh Token | 자동 갱신 여부 |
|---|---|---|---|

| Threads | 60일 | 60일 | ✅ 자동 갱신 |
| X | 2시간 | 6개월 | ✅ 자동 갱신 |

#### 토큰 자동 갱신 시스템

```
[Celery Beat 주기 작업 — 매일 오전 3시 실행]
          │
          ▼
모든 클라이언트 토큰 만료일 일괄 점검
          │
    ┌─────┴──────┐
    ▼            ▼
만료 7일 전    만료 당일 임박
    │            │
    ▼            ▼
자동 갱신 시도   긴급 갱신 시도
(Refresh Token)  (Refresh Token)
    │            │
    ├─ 성공 ─────┤
    │            │
    ▼            ▼
새 Token 저장   갱신 실패
만료일 업데이트      │
                     ▼
             담당자 알림 발송
             + 클라이언트 재인증 요청
```

#### 토큰 갱신 실패 시 처리 (핵심)

**갱신이 실패하는 경우:**
1. 클라이언트가 **비밀번호를 변경**했을 때
2. 클라이언트가 **앱 권한을 직접 취소**했을 때
3. 플랫폼 **정책 변경**으로 권한이 무효화됐을 때
4. Refresh Token도 **만료**됐을 때 (링크드인 등)

**갱신 실패 시 자동 대응:**

```
토큰 갱신 실패 감지
        │
        ▼
① 즉시 처리
   - 해당 채널 발행 큐 일시 중단
   - 예약된 게시물 발행 보류 (삭제 X, 대기 상태로 전환)
   - 댓글/DM 수집 중단

② 알림 발송 (3단계)
   - 내부 담당자 텔레그램 즉시 알림:
     "⚠️ [클라이언트A] 인스타그램 토큰 만료
      재연동이 필요합니다. 예약 게시물 3건 보류 중"
   
   - 클라이언트 이메일 발송 (외부 승인 방식인 경우):
     "SNS 계정 재연동 요청"
     + 재연동 버튼 링크 포함
   
   - 24시간 내 미해결 시 Admin 에스컬레이션 알림

③ 재연동 안내 페이지
   - 담당자가 링크 클릭 → 해당 채널 재인증 팝업 바로 오픈
   - 재연동 완료 시 보류 게시물 자동 재개 여부 확인:
     "보류된 게시물 3건을 지금 바로 발행할까요?"
     [즉시 발행] [예약 유지] [취소]
```

| 항목 | 방식 |

### 4-1-1. 클라이언트 온보딩 (워크스페이스 초기 설정)

> 새 클라이언트 워크스페이스 생성 시 아래 온보딩 단계를 순서대로 진행합니다.
> 온보딩 완료 후 AI가 이 설정을 학습하여 콘텐츠 생성/운영 전략에 반영합니다.

---

#### Step 1. 계정 유형 선택

```
이 SNS 계정의 운영 목적은 무엇인가요?

  ○ 공공기관 / 정부·지자체
  ○ 제품 중심 (커머스/쇼핑)
  ○ 브랜드 운영 (인지도/팬덤 구축)
  ○ 서비스 홍보 (B2B/B2C 서비스)
  ○ 이벤트·캠페인 전용
  ○ 기타 (직접 입력)
```

> 계정 유형에 따라 콘텐츠 전략 템플릿, KPI 기준, 해시태그 방식, 톤앤매너 기본값이 달라집니다.

---

#### Step 2. 톤앤매너 설정

**2-1. 기본 톤 선택 (복수 선택 가능)**

| 톤 | 설명 | 예시 |
|---|---|---|
| 공식적/신뢰감 | 기관·기업 공식 계정 느낌 | "안내드립니다" |
| 친근/캐주얼 | MZ세대 타깃, 편한 말투 | "같이 해봐요!" |
| 전문적/정보형 | 업계 지식 전달 중심 | "데이터 기반 인사이트" |
| 감성/스토리텔링 | 공감·감동 중심 | "이런 순간, 있으셨나요?" |
| 유머/위트 | 가볍고 재미있는 컨셉 | "...라고 하더라구요 🫠" |
| 프리미엄/럭셔리 | 고급 브랜드 이미지 | 짧고 절제된 문장 |

**2-2. 금지 표현 / 회피 키워드 입력**
- 예: 경쟁사 언급 금지, 특정 단어 사용 금지, 정치적 표현 금지 등

**2-3. 선호 이모지 사용 여부**
- 사용 / 미사용 / 제한적 사용 (1개 이하/문장)

**2-4. 해시태그 전략**
- 많이 사용 (인스타 30개 풀) / 적당히 (5~10개) / 최소화 (3개 이하) / 미사용

---

#### Step 3. 벤치마킹 채널 등록

> 참고하고 싶은 SNS 계정을 입력하면 AI가 해당 채널의 콘텐츠 스타일을 분석하여 유사한 컨셉으로 운영 전략을 수립합니다.

**입력 항목:**
| 필드 | 설명 |
|---|---|
| 채널 URL 또는 @계정명 | 인스타/X/유튜브/페북/스레드 |
| 플랫폼 선택 | 어느 SNS인지 |
| 벤치마킹 목적 | 톤앤매너 / 콘텐츠 형식 / 해시태그 / 전체 |
| 메모 | "이 채널처럼 캐릭터 중심으로 운영하고 싶다" 등 자유 입력 |

- 최대 5개 채널 등록 가능
- 등록 후 AI가 최근 30개 게시물 스타일 자동 분석
- 분석 결과: 주요 톤앤매너 / 게시 패턴 / 해시태그 방식 / 콘텐츠 포맷 요약

---

#### Step 4. 클라이언트 담당자 의견 수렴 (선택)

> 외부 승인 방식인 경우, 온보딩 단계에서 클라이언트 담당자에게 톤앤매너 확인 설문을 이메일로 발송할 수 있습니다.

**설문 항목 (클라이언트에게 이메일 발송):**
1. 우리 브랜드를 한 단어로 표현한다면?
2. 절대 하면 안 되는 표현이 있나요?
3. 좋아하는 SNS 계정이 있다면 공유해주세요.
4. 타깃 고객층은 어떤 사람들인가요?
5. 추가로 전달하고 싶은 내용

- 설문 응답은 워크스페이스 설정에 자동 저장
- AI 콘텐츠 생성 시 참조 컨텍스트로 활용

---

#### Step 5. AI 학습 완료 및 전략 보고서 생성

온보딩 완료 후 AI가 자동으로 **채널 운영 전략서** 초안을 생성:

```
📋 채널 운영 전략서 (AI 초안)
─────────────────────────
계정 유형: 브랜드 운영
톤앤매너: 친근/캐주얼 + 감성/스토리텔링
벤치마킹: @example_brand 스타일 반영

권장 콘텐츠 믹스:
  - 제품/서비스 소개: 30%
  - 일상/스토리텔링: 40%
  - 이벤트/참여: 20%
  - 정보형 콘텐츠: 10%

권장 발행 주기: 주 4~5회 (화/목/토 집중)
권장 해시태그: 10~15개 혼합
```

- 전략서는 워크스페이스 설정에 저장
- 언제든 수정 가능
- 콘텐츠 AI 생성 시 이 전략서를 기본 컨텍스트로 사용

### 4-1-C. 콘텐츠 수정 / 삭제 / 재발행 / 휴지통

#### FR-06. 콘텐츠 수정

| 상태 | 수정 가능 여부 | 조건 |
|---|---|---|
| 초안(draft) | ✅ 자유 수정 | 제한 없음 |
| 승인 대기 중 | ✅ 수정 가능 | 수정 시 승인 요청 자동 취소 → 재요청 필요 |
| 승인 완료 | ✅ 수정 가능 | 수정 시 승인 상태 초기화 → 재승인 필요 |
| 예약 발행 대기 | ✅ 수정 가능 | 발행 1시간 전까지만 수정 가능 (이후 잠금) |
| 발행 완료 | ⚠️ 제한적 수정 | 텍스트만 수정 가능 (이미지 교체 불가), 플랫폼 정책 제약 있음 |

**발행 완료 콘텐츠 수정 시:**
- 수정 내용이 실제 SNS 게시물에 반영되려면 플랫폼 API 재호출 필요
- 인스타그램: 캡션만 수정 가능 (이미지 교체 불가)
- X: 수정 불가 (삭제 후 재발행 방식)
- 수정 이력 자동 기록 (버전 관리)

#### FR-07. 콘텐츠 삭제 (휴지통)

**삭제 플로우:**
```
삭제 버튼 클릭
  → 확인 모달: "휴지통으로 이동하시겠습니까?"
  → 확인 → 휴지통 이동 (status: trashed)
  → 30일 대기
  → 30일 경과 → 자동 완전 삭제
```

**휴지통 기능:**
- [콘텐츠 > 휴지통] 탭에서 삭제된 항목 목록 조회
- 삭제일 + 자동 완전 삭제 예정일 표시
- **복원**: 휴지통에서 초안(draft) 상태로 복원
- **즉시 영구 삭제**: 30일 기다리지 않고 즉시 완전 삭제
- **휴지통 비우기**: 전체 항목 즉시 영구 삭제 (확인 모달 필수)

**발행된 콘텐츠 삭제 시:**
- 플랫폼에서도 삭제할지 선택 옵션
  - "플랫폼에서도 삭제" → SNS 게시물 삭제 API 호출 후 휴지통 이동
  - "플랫폼은 유지" → 플랫폼 게시물 유지, 내부 데이터만 휴지통 이동

#### FR-08. 콘텐츠 재발행

**재발행 가능 조건:** 발행 완료 상태의 콘텐츠

**재발행 유형:**
| 유형 | 설명 |
|---|---|
| 동일 내용 재발행 | 발행 완료 콘텐츠를 그대로 다시 예약/발행 |
| 수정 후 재발행 | 내용 수정 → 승인 → 재발행 |
| 채널 추가 재발행 | 기존 발행 채널 외 추가 채널에 발행 |
| 시즌 재발행 | 지난 콘텐츠를 새 날짜로 재예약 |

#### FR-09. 게시물 유형 선택 (콘텐츠 작성 첫 단계)

콘텐츠 작성 시작 시 **게시물 유형을 먼저 선택** → 유형에 따라 에디터 UI와 AI 생성 방식이 달라집니다.

---

**유형 1. 카드뉴스**

> 최소 5장 이상. 톤앤매너 세트 선택 → 전체 제작 2단계 방식으로 진행합니다.

**Step 1. 톤앤매너 컨셉 3세트 제안 (AI 자동 생성)**

주제/키워드 입력 후 "컨셉 제안" 클릭 → AI가 3가지 스타일로 컨셉 초안 생성:

```
[세트 A] 정보형 / 깔끔
  - 대표 카피: "알고 계셨나요? 탄소배출 줄이는 5가지 방법"
  - 톤: 신뢰감 있는 정보 전달, 수치 강조
  - 컬러 방향: 화이트 + 딥네이비
  - 이미지 방향: 인포그래픽, 아이콘 중심
  - 미리보기: 1번 슬라이드 썸네일 (Fal.ai 생성)

[세트 B] 감성형 / 스토리텔링
  - 대표 카피: "오늘 당신의 선택이 내일을 바꿉니다"
  - 톤: 공감과 감동, 따뜻한 말투
  - 컬러 방향: 아이보리 + 그린 계열
  - 이미지 방향: 자연 배경, 사람 중심
  - 미리보기: 1번 슬라이드 썸네일

[세트 C] 임팩트형 / 훅
  - 대표 카피: "당신만 모르는 탄소중립의 진실"
  - 톤: 강한 훅, 궁금증 유발
  - 컬러 방향: 블랙 + 포인트 컬러
  - 이미지 방향: 대형 타이포그래피, 대비 강조
  - 미리보기: 1번 슬라이드 썸네일
```

**Step 2. 세트 선택 → 전체 카드뉴스 제작**

- 세트 선택 후 장 수 설정: 5 / 6 / 8 / 10장 (기본 6장)
- AI가 선택된 톤앤매너 + 장 수에 맞게 전체 스토리 구성
- 각 장별 카피 + 이미지 자동 생성 (Fal.ai Nano Banana)
- Canva API로 브랜드 템플릿에 자동 조립

**Step 3. 미리보기 — 2장 연속 뷰어**

```
┌──────────────┬──────────────┐
│   슬라이드 1  │   슬라이드 2  │  ← 2장씩 나란히 표시
│   (표지)     │  (문제제기)   │
└──────────────┴──────────────┘
        < 이전          다음 >

[전체 슬라이드 썸네일 바]
 ①  ②  ③  ④  ⑤  ⑥
 ●  ●  ○  ○  ○  ○   ← 현재 위치 표시
```

- **2장 나란히 보기** (기본) — 앞뒤 흐름 확인
- **1장 단독 보기** 전환 버튼
- **전체 슬라이드 썸네일 바** 하단 고정 — 클릭으로 해당 장으로 이동
- **슬라이드 넘기기**: 키보드 방향키 / 스와이프 지원
- 각 장 위에 **직접 수정** 버튼 (카피 수정 / 이미지 재생성 / 레이아웃 변경)
- **연속성 체크**: AI가 장 간 스토리 흐름 자동 검토 → 어색한 전환 경고

**스토리 구성 원칙 (AI 자동 적용):**
```
1장: 훅 / 표지 (시선 끌기)
2장: 문제 제기 / 공감
3~N-1장: 핵심 내용 (장당 1포인트)
N장: CTA / 마무리 (행동 유도)
```

- 플랫폼: 인스타그램 / 페이스북 / 스레드 / 카카오채널

#### FR-13-B. 콘텐츠 수정 방식 — 대화형 + 직접 편집 혼합

---

**방식 1. 대화형 수정 (AI 채팅)**

화면 우측에 AI 채팅 패널 상시 노출:

```
┌─────────────────────┬──────────────────┐
│   콘텐츠 미리보기    │   AI 채팅 패널   │
│                     │                  │
│  [슬라이드 1] [2]   │ 💬 어떻게 수정   │
│                     │    할까요?       │
│  ① ② ③ ④ ⑤ ⑥    │                  │
│                     │ > 3번 슬라이드   │
│                     │   톤 더 친근하게 │
│                     │                  │
│                     │ > 전체 해시태그  │
│                     │   다시 짜줘      │
│                     │                  │
│                     │ > 이미지 더 밝게 │
│                     │   재생성해줘     │
└─────────────────────┴──────────────────┘
```

**대화형 수정 가능 범위:**
| 명령 예시 | 처리 |
|---|---|
| "3번 슬라이드 카피 더 짧게" | 해당 슬라이드 카피만 재생성 |
| "전체 톤 더 친근하게 바꿔줘" | 전체 카피 톤 일괄 변경 |
| "이미지 색감 더 따뜻하게" | 해당 이미지 Fal.ai 재생성 |
| "해시태그 10개로 줄여줘" | 해시태그만 재추천 |
| "2번이랑 3번 순서 바꿔줘" | 슬라이드 순서 변경 |
| "마지막 장 CTA 더 강하게" | 마지막 슬라이드만 재생성 |
| "전체 다시 만들어줘" | 전체 초안 재생성 |

- 수정 후 즉시 미리보기 반영
- 수정 이력 채팅창에 타임라인으로 기록
- "이전으로 되돌리기" 버튼 (최대 10단계)

---

**방식 2. 직접 인라인 편집**

슬라이드 위에서 직접 클릭하여 편집:

- **텍스트 클릭** → 인라인 텍스트 에디터 활성화 (바로 타이핑)
- **이미지 클릭** → 이미지 액션 메뉴
  - 재생성 (AI) / 직접 업로드 / 소재 라이브러리에서 선택
- **레이아웃 클릭** → 레이아웃 변경 드롭다운
- **슬라이드 드래그** → 순서 변경

#### FR-22. 외부 승인 플로우 (승인자: 클라이언트)

```
Editor 작성 → "클라이언트 승인 요청" 클릭
  → 클라이언트 이메일로 시안 발송
  → 클라이언트가 이메일 내 링크 클릭
  → 전용 승인 페이지에서 시안 확인
  → 승인 or 수정 요청 (코멘트 입력)
  → 승인 시 → 내부 담당자 알림 → 자동 예약발행 대기
  → 수정 요청 시 → Editor 알림 + 코멘트 전달
```

**클라이언트 승인 이메일 구성:**
- 제목: `[에임탑] {클라이언트명} SNS 콘텐츠 시안 승인 요청`
- 내용: 플랫폼별 콘텐츠 미리보기 이미지 + 텍스트
- 버튼: **승인하기** / **수정 요청**
- 링크 유효기간: 7일 (만료 시 재발송 가능)
- 로그인 불필요 (토큰 기반 1회용 링크)

**클라이언트 전용 승인 페이지:**
- 로그인 없이 접근 가능 (보안 토큰 URL)
- 콘텐츠 미리보기 (채널별 탭)
- 승인 버튼 / 수정 요청 + 코멘트 입력
- 승인 완료 시 "승인이 완료되었습니다" 화면 표시

#### FR-60-A. 댓글 활성화 전략

**1. 질문형 CTA 자동 삽입**
- 카피 생성 시 AI가 댓글 유도 질문을 마지막 문장에 자동 포함
- 예: "여러분은 어떻게 생각하시나요? 댓글로 알려주세요 👇"
- 계정 유형별 CTA 템플릿 자동 선택
  - 공공기관: "여러분의 의견을 들려주세요"
  - 제품: "이 제품 써보셨나요?"
  - 브랜드: "나만의 스타일은? 태그해주세요"

**2. 댓글 골든타임 알림**
```
발행 후 30분~2시간이 댓글 활성화 골든타임
  → 담당자에게 알림: "지금 게시물에 댓글 달아주세요!
     초기 댓글이 알고리즘 노출을 높입니다"
  → 브랜드 계정 담당자가 직접 첫 댓글 달기 유도
  → 첫 댓글 예시 자동 생성 (AI)
```

**3. 댓글 이벤트 자동 설정**
- 이벤트 게시물 발행 시 자동으로:
  - "댓글 + 팔로우 = 이벤트 응모"
  - 당첨자 댓글 추첨 기능 (댓글 목록 → 무작위 당첨)
  - 당첨 발표 게시물 자동 생성

**4. 댓글 답글 자동화 (기존 FR-43 확장)**
- 빠른 답글 템플릿: "감사합니다 😊", "맞아요!", "좋은 의견이에요!"
- AI가 댓글 내용 분석 → 맞춤 답글 초안 생성
- **답글 속도 목표**: 1시간 이내 답변 (답글 속도 = 알고리즘 우대)
- 미답변 댓글 30분 경과 시 담당자 알림 재발송

**5. 핀 댓글 관리**
- 인스타/유튜브 핀 댓글 설정 자동화
- 브랜드 공지사항, 이벤트 안내를 핀 댓글로 고정
- 게시물 발행 시 핀 댓글 내용 함께 입력 → 자동 등록

#### FR-60-B. 팔로워 성장 전략

**1. 팔로우 유도 콘텐츠 믹스 관리**
```
팔로워 성장에 효과적인 콘텐츠 비율 자동 관리:

저장형 콘텐츠  (인포그래픽, 유용한 정보)  → 신규 도달↑
공감형 콘텐츠  (스토리텔링, 감성)         → 팔로우↑
참여형 콘텐츠  (이벤트, 질문)             → 댓글↑
브랜드 콘텐츠  (제품, 서비스 소개)        → 전환↑

월간 플랜 생성 시 위 비율 자동 반영
```

**2. 해시태그 전략 고도화**
- 대형 (#1억+ 게시물) / 중형 (#100만~) / 소형 (#10만 이하) 혼합 자동 설정
- 소형 해시태그에서 상위 노출 확률 극대화
- 경쟁도 낮은 틈새 해시태그 AI 자동 발굴
- 해시태그 성과 추적 (어떤 태그에서 도달이 왔는지)

**3. 계정 간 교류 (Engagement Pod)**
- 에임탑이 관리하는 여러 클라이언트 계정 간 상호 좋아요/댓글 유도
- 관련 업종 계정 간 협업 게시물 기획 알림
- ⚠️ 각 플랫폼 정책 내에서만 운영

**4. 인스타그램 릴스 최적화**
- 릴스 첫 3초 훅 강도 자동 점수화 (AI)
- 자막 스타일 A/B 테스트 (굵은 자막 vs 얇은 자막)
- 릴스 커버 이미지 최적화 (썸네일 클릭률 영향)
- 오디오 트렌드 추천 (현재 인기 BGM 자동 제안)

**5. 스토리 활용 전략**
- 스토리 → 피드 게시물 유도 스와이프업 링크
- 하이라이트 구성 가이드 (브랜드 스토리 아카이브)
- 스토리 설문/퀴즈로 팔로워 참여 유도
- 스토리 조회수 → 팔로우 전환율 추적

**6. 팔로워 증감 알림**
```
팔로워 변화 실시간 모니터링:
  +100명 돌파 → 담당자 알림 (🎉)
  하루 -50명 이상 언팔 → 즉시 알림 + 원인 분석
  팔로워 성장 정체 2주 이상 → 전략 재검토 알림
```

**7. 콘텐츠 발행 최적 시간 준수**
- 팔로워 활성 시간대 분석 → 발행 시간 자동 추천
- 플랫폼별 최적 시간 (인스타: 화/목/금 오전 9시~11시 등)
- 경쟁 계정 발행 시간 겹침 회피 (가능한 범위 내)

| 관계 상태 | 미접촉/접촉중/파트너십/완료 |

**2. 자동 모니터링 및 교류 액션**

```
등록된 타깃 계정 게시물 실시간 모니터링
          │
          ▼
  관련 게시물 감지 시 담당자에게 알림
  ─────────────────────────────
  "파트너 계정 @xxx가 새 게시물 발행"
  "우리 브랜드 관련 주제: 친환경 식품"
  ─────────────────────────────

    ┌─────┴─────┐
    ▼           ▼

  (1클릭)     AI 초안 생성
```

- 업종 트렌드 해시태그 급상승

**교류 액션 (모두 담당자 승인 후 실행):**
| 액션 | 설명 |
|---|---|
| 좋아요 | 1클릭 승인 후 자동 처리 |
| 댓글 | AI 초안 → 수정 → 승인 → 발송 |
| RT/공유 | 승인 후 자동 처리 |
| 멘션 | 게시물 작성 시 타깃 계정 태그 추천 |
| DM | AI 초안 작성 → 담당자 발송 (협업 제안 등) |

## 4-8. 로그인 / 인증 시스템

### FR-100. 로그인

- 이메일 + 비밀번호 기반 로그인
- 자동 로그인 (Remember me, 30일)
- 세션 만료: 비활성 2시간 후 자동 로그아웃
- 만료 10분 전 "세션이 곧 만료됩니다" 경고 → 연장 버튼

### FR-101. 비밀번호 관리

- 비밀번호 찾기: 이메일로 재설정 링크 발송 (유효기간 1시간)
- 비밀번호 변경: 현재 비밀번호 확인 후 변경
- 비밀번호 규칙: 8자 이상, 영문+숫자+특수문자 포함

### FR-102. 2단계 인증 (2FA, 선택)

- Admin 계정은 2FA 강제 적용 (선택 아닌 필수)
- Editor / Approver / Viewer: 선택 적용
- 방식: 이메일 OTP (6자리, 5분 유효) 또는 TOTP 앱 (Google Authenticator)

### FR-103. 보안 이벤트 로그

- 로그인 성공/실패 기록 (IP, 시간, 기기)
- 5회 연속 로그인 실패 → 30분 계정 잠금 + Admin 알림
- 새 기기/IP 로그인 시 이메일 알림

### FR-60. 발행 아카이브

- 발행 완료된 모든 게시물이 자동으로 라이브러리에 저장
- 클라이언트별 / 채널별 / 유형별 / 기간별 필터
- 검색 (키워드, 해시태그, 날짜)

### FR-61. 게시물 재사용 / 복제

- **복제**: 기존 게시물 그대로 복사 → 수정 후 재발행
- **재활용**: 성과 좋았던 게시물 → "유사 콘텐츠 생성" 버튼 → AI가 비슷한 톤으로 새 초안 생성
- **시즌 재활용**: 작년 동기 게시물 불러오기 (연간 이벤트 반복 활용)

### FR-62. 콘텐츠 태그 / 분류

- 담당자가 게시물에 태그 직접 추가 (예: #신제품, #이벤트, #인포그래픽)
- 태그 기반 묶음 조회
- 클라이언트별 태그 사전 관리

## 11. 월간 콘텐츠 플랜

### FR-80. 월간 플랜 작성

- 한 달치 게시 일정을 미리 기획하는 기능
- 캘린더에서 날짜 선택 → 유형/주제 입력 → 플랜 등록
- AI 자동 플랜 제안: 월 목표 게시 수 입력 → AI가 날짜별 유형/주제 초안 생성

**AI 플랜 제안 기준:**
- 계정 유형별 콘텐츠 믹스 비율 (운영 전략서 기반)
- 공휴일 / 시즌 이벤트 자동 반영
- 최적 발행 요일/시간 배분

### FR-81. 일괄 승인

- 월간 플랜 작성 후 클라이언트에게 **한 번에 전체 승인 요청** 가능
- 클라이언트가 월간 플랜 요약본 확인 후 전체 승인 또는 개별 항목 수정 요청
- 승인 완료 시 각 게시물 자동 예약 등록

### FR-82. 플랜 vs 실적 비교

- 계획된 게시 수 vs 실제 발행 수
- 미발행 항목 자동 감지 및 알림
- 월말 자동 플랜 달성률 리포트

## 14. 에러 / 예외 처리

### 발행 실패 처리

```
발행 시도
  → 성공: 발행 완료 알림
  → 실패: 자동 재시도 (3회, 5분 간격)
       → 3회 모두 실패: 담당자 즉시 알림
       → 실패 원인 기록 (토큰 만료 / API 한도 / 네트워크 등)
       → 수동 재발행 버튼 노출
```

### SNS 토큰 만료 대응

| 상황 | 처리 |
|---|---|
| 만료 7일 전 | Admin에게 사전 경고 알림 |
| 만료 1일 전 | 긴급 알림 (텔레그램 + 이메일) |
| 만료 시 | 해당 채널 발행 일시 중단 + 재인증 유도 |
| 재인증 완료 | 대기 중이던 예약 게시물 자동 재개 |

### API 사용량 한도 대응

| 상황 | 처리 |
|---|---|
| 80% 도달 | Admin 알림 |
| 100% 초과 | 발행 큐 대기 + 담당자 알림 |
| 다음 날 리셋 후 | 대기 게시물 자동 발행 |

### 콘텐츠 오류 감지 (AI 자동)

- 맞춤법 오류 감지 → 수정 제안
- 금지 키워드 포함 → 발행 전 경고
- 링크 유효성 검사 (첨부 URL 404 감지)
- 이미지 해상도 부족 → 경고 표시
- 글자 수 초과 → 채널별 제한 초과 시 경고

```
  → Canva API: 브랜드 템플릿 선택
  → 각 슬라이드에 텍스트 자동 삽입
  → 미리보기 확인 → 수정 → 승인 요청
```

**릴스/숏폼 제작 플로우:**
```
영상 컨셉 입력
  → Claude: 씬 구성 + 자막 스크립트 생성
  → 담당자가 영상 촬영 또는 기존 클립 업로드
  → 자막 타임코드 편집
  → 플랫폼 업로드 (upload-post API)
```

**AI 이미지 + 카피 통합 제작:**
```
주제/키워드 입력
  → Claude: 카피 + 이미지 프롬프트 동시 생성
  → Gemini Image: 프롬프트 기반 이미지 생성 (3가지 옵션)
  → 이미지 선택 + 카피 확정
  → Canva: 텍스트/로고 오버레이
  → 최종 결과물 다운로드
```

```python
def generate_card_news(client_id, topic, slide_count):

    refs = get_top_references(tone_profile, count=3)

        context=strategy + tone_profile,
        topic=topic,

        image = fal.generate(model="nano-banana-pro", 
                            prompt=prompt, 
                            references=refs)
        save_prompt_log(id, i, prompt, image)

    save_json(f"clients/{client_id}/contents/{id}/content.json",
```

```
    → 담당자 완료 알림
            │
            ▼
    [발행 후 자동]
    댓글 모니터링 시작
    성과 데이터 수집 (24h 후, 7d 후)
    월말 리포트에 자동 반영
```
